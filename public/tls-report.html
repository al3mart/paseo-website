Paseo Asset Hub — WSS TLS Audit

Full inventory of peers discovered on the Paseo Asset Hub network. For each node exposing a WebSocket Secure (wss) endpoint, we verified the TLS certificate — a requirement for smoldot light clients running in browsers.

Date: 2026-04-16 Chain: asset-hub-paseo Para ID: 1000 Discovery: 90s Kademlia DHT crawl

Methodology

How this audit was performed

  1. We loaded the paseo-asset-hub.smol.json and paseo.raw.smol.json chain specs and started a smoldot light client in Node.js.
  2. Smoldot connects to the bootnodes listed in the chain spec and joins the libp2p network. It then performs Kademlia DHT peer discovery — random walks through the distributed hash table to find other nodes.
  3. We captured all multiaddrs announced by discovered peers from smoldot’s debug logs for 90 seconds, yielding 127 unique peers.
  4. For every endpoint with a /wss/ segment in its multiaddr, we verified the TLS certificate: 
    echo Q | timeout 5 openssl s_client \
  -connect <host>:<port> \
  -servername <host>
    checking the verify return code, issuer (CA), and expiration date.
  5. Results were classified as valid, expiring (within 7 days), expired, or unreachable.

Reproduce it yourself

The audit tooling is open source. You need Bun installed, then:

git clone https://github.com/paseo-network/paseo-tls-audit.git
cd paseo-tls-audit
bun install
bun run audit

This runs three steps: discover (smoldot DHT crawl) → check-tls (certificate verification) → report (HTML generation). The final report is written to output/report.html.

You can also run each step individually — see the repo README for details.
